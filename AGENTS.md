# Agent working rules

## Mission
Improve the visual design quality of the existing HTML mockups with a stronger, more human visual identity.

## Priorities
1. Visual personality
2. Strong hierarchy
3. Better rhythm and spacing
4. Composition quality
5. Interaction polish
6. Responsive consistency
7. Clean code

## Design principles
- Avoid generic SaaS layouts
- Avoid repetitive card grids with identical spacing
- Avoid uniform section patterns
- Prefer editorial composition when relevant
- Use asymmetry carefully to create tension
- Increase contrast in typography scale
- Make CTA blocks feel intentional, not default
- Create 1 to 3 recurring signature details across the UI
- Preserve clarity and usability
- Keep motion subtle and purposeful
- Make every section feel designed, not auto-generated

## CSS expectations
- Use design tokens where possible
- Normalize spacing scale
- Normalize border radius scale
- Normalize shadow language
- Normalize container widths
- Normalize interactive states
- Keep responsive behavior robust

## Output expectations
When asked to refine a page:
- first analyze the current weaknesses
- then propose a design direction
- then implement the changes
- then explain what changed and why
- then list remaining weak points if any
